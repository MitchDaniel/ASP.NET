Задание 1
Создайте новый проект Web API.
Удалите, если имеется контроллер WeatherForecastController, созданный по умолчанию.
В папке Controllers создайте OrdersController на основе ControllerBase.
Установите атрибут [ApiController] и маршрут вида api/[controller].
Добавьте два действия-заглушки: GET /api/orders и GET /api/orders/{id} (на первом шаге — возврат 200 OK).
Запустите приложение и убедитесь, что оба маршрута отвечают 200 OK.

Задание 2
Создайте папку Models и класс Order с полями: Id (int), Number (string), Total (decimal), CreatedAt (DateTime).
В OrdersController подготовьте список заказов в памяти (не менее трёх записей) с уникальными Id и разными значениями полей.
Обновите GET /api/orders, чтобы возвращался список заказов.
Обновите GET /api/orders/{id}, чтобы при отсутствии записи возвращался 404 Not Found с текстом Order not found.
Установите ограничение маршрута для параметра {id} как целого числа.
Проверьте ответы и статусы 200 (существующий id) и 404 (несуществующий id).

Задание 3
Создайте DTO для создания/обновления заказа (минимум обязательные поля: Number, Total).
Реализуйте POST /api/orders с чтением тела запроса из body:
При пустом/отсутствующем обязательном поле — 400 Bad Request с понятным сообщением.
При успешном создании назначьте новый Id, добавьте запись в коллекцию и верните 201 Created с заголовком Location, указывающим на GET /api/orders/{id}.
Реализуйте PUT /api/orders/{id}:
Если заказ не найден — 404 Not Found.
При некорректных данных во входном DTO — 400 Bad Request.
При успехе — 200 OK с обновлённым объектом.
Проверьте сценарии: успешный POST, успешный PUT, PUT по несуществующему id (ожидается 404).

Задание 4
Добавьте DELETE /api/orders/{id}:
Если заказ не найден — 404 Not Found.
При успешном удалении — 204 No Content.
Добавьте GET /api/orders/search с параметрами строки запроса:
Обязательный number — подстрочный поиск без учёта регистра.
Необязательный minTotal — фильтр по сумме (decimal, включает границу).
При отсутствии обязательного number — 400 Bad Request с понятным текстом.
При корректных параметрах — 200 OK и список результатов (допускается пустой список).
Выполните последовательный сценарий в любом HTTP-клиенте:
Создать заказ (POST) → ожидается 201 Created и новый Id.
Прочитать по Id (GET /{id}) → 200 OK.
Обновить (PUT /{id}) → 200 OK.
Найти (GET /search?number=…&minTotal=…) → 200 OK.
Удалить (DELETE /{id}) → 204 No Content.
Повторно прочитать тот же Id → 404 Not Found.
