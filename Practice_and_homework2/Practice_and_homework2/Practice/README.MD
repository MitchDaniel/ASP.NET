#Задание 1
Создайте новый проект Web API.
Удалите, если имеется контроллер WeatherForecastController, созданный по умолчанию.
В папке Controllers создайте ItemsController на основе ControllerBase.
Установите атрибут [ApiController] и маршрут вида api/[controller].
Добавьте два действия: GET /api/items и GET /api/items/{id}, на первом этапе возвращающие 200 OK с заглушками.
Запустите приложение и убедитесь, что оба маршрута отвечают 200 OK.

#Задание 2
Создайте папку Models и класс Item с полями Id и Name.
Внутри ItemsController подготовьте список элементов в памяти минимум из трёх записей с Id = 1, 2, 3 и разными значениями Name.
Обновите действие GET /api/items, чтобы возвращался список элементов.
Обновите действие GET /api/items/{id}, чтобы при отсутствии записи возвращался 404 Not Found с текстом Item not found.
Установите ограничение маршрута для параметра {id} как целого числа.
Проверьте ответы по маршрутам и статусы 200 и 404.

#Задание 3
В ItemsController добавьте маршрут GET /api/items/search с обязательным параметром строки запроса name.
При отсутствии параметра name возвращайте 400 Bad Request с понятным сообщением об ошибке.
При наличии параметра name верните отфильтрованный список по вхождению имени без учёта регистра.
Если совпадений нет, верните пустой список со статусом 200 OK.
Проверьте результаты по адресам вида /api/items/search?name=pen и /api/items/search (без параметра).

#Задание 4
Добавьте действие POST /api/items для создания элемента.
Принимает объект с обязательным текстовым полем имени.
- При пустом имени возвращает 400 Bad Request с сообщением об ошибке.
- При успешном создании добавляет запись в список с новым Id и возвращает 201 Created со ссылкой на GET /api/items/{id}.
Добавьте действие PUT /api/items/{id} для обновления существующего элемента.
Параметр {id} — целое число (ограничение маршрута).
- При отсутствии элемента возвращает 404 Not Found.
- При пустом имени в запросе возвращает 400 Bad Request.
- При успешном обновлении возвращает 200 OK с обновлённым объектом.
Добавьте действие DELETE /api/items/{id} для удаления элемента.
Параметр {id} — целое число (ограничение маршрута).
- При отсутствии элемента возвращает 404 Not Found.
- При успешном удалении возвращает 204 No Content.
Выполните мини-сценарий проверки (любой HTTP-клиент).
- Создание элемента -> ожидается 201 Created и новый Id.
- Чтение по-новому Id -> ожидается 200 OK и корректные данные.
- Обновление имени по этому Id -> ожидается 200 OK и обновлённые данные.
- Удаление по этому Id -> ожидается 204 No Content.
- Повторное чтение того же Id -> ожидается 404 Not Found.
